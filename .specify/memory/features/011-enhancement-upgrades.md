## Feature Description

The Enhancement & Upgrade Planning feature is specifically designed for "Existing Assessment" projects — those where a consultant is evaluating an operational broiler house rather than designing a new one. It is the core value proposition for the assessment workflow, enabling consultants to generate data-driven upgrade recommendations and present them as structured investment proposals to farmers.

After the consultant has entered the existing house's dimensions, current equipment, and received its risk/welfare scores, this feature automatically analyzes the assessment results and generates a prioritized list of upgrade recommendations. Recommendations are categorized by urgency: Critical (e.g., ventilation capacity must be increased from 12,000 to 28,000 CFM), High (e.g., wall insulation upgrade from R-12 to R-28), and Medium (e.g., LED lighting retrofit from fluorescent T8 tubes). Each recommendation includes the proposed change, estimated cost, projected ROI timeline, and the specific metric improvement it would achieve (e.g., "Heat loss reduction: 45%", "Energy efficiency: +60%").

The key visual element is the **Before vs. After comparison**. For each recommended enhancement, the system shows side-by-side panels: the current state metrics alongside the projected metrics after the upgrade is applied. This comparison spans financial impact (cost savings, revenue increase), risk scores (mortality risk reduction, compliance improvement), and environmental performance. The contrast makes the business case immediately tangible for farmers who may not understand the technical details.

Users can check/uncheck individual enhancements to build a custom investment proposal — perhaps the farmer wants to prioritize ventilation this year and defer insulation to next year. As selections change, the summary totals at the bottom update: total estimated cost vs. projected annual savings, and the combined impact on compliance and risk scores.

This feature appears as Dashboard Tab 4, visible only for "Existing Assessment" projects. It builds directly on top of the financial, risk, and blueprint data from prior features.

## Acceptance Criteria Summary

- [ ] System auto-generates prioritized upgrade recommendations (Critical, High, Medium) based on assessment data
- [ ] Each recommendation shows proposed change, estimated cost, ROI timeline, and metric improvement
- [ ] Side-by-side Before vs. After comparison panels display current vs. projected metrics for each enhancement
- [ ] Users can check/uncheck individual enhancements to build a custom investment proposal
- [ ] Summary totals (total cost, annual savings, compliance impact) update dynamically as selections change
- [ ] Enhancement tab is visible only for "Existing Assessment" projects, not "New Installation" projects
- [ ] Recommendations cover ventilation, insulation, and equipment categories at minimum

## UI Components (if applicable)

- **EnhancementTab**: Dashboard Tab 4 composable with recommendation list, comparisons, and action bar
- **EnhancementRecommendationCard**: Priority-colored card (Critical=Red, High=Orange, Medium=Yellow) with change description, cost, ROI, and metric improvement
- **BeforeAfterComparisonPanel**: Side-by-side boxes showing "Current State" vs. "Proposed State" values
- **EnhancementCheckbox**: Toggle control for including/excluding enhancements from the investment proposal
- **InvestmentSummaryBar**: Bottom bar showing total estimated cost vs. annual savings with apply/export actions

## Technical Hints

- This feature is only relevant for "Existing Assessment" project types — the navigation should conditionally show/hide the Enhancement tab based on `project.type`
- Enhancement recommendations are generated by comparing current metrics against breed-optimal standards and EU welfare thresholds — the algorithm should be a domain use case with no UI dependencies
- Before/After projections require re-running financial and risk calculations with modified parameters — consider a "what-if" variant of the existing calculation use cases
- Constitution Article 2.5 requires documentation of the recommendation prioritization algorithm
